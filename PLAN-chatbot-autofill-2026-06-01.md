# PLAN — чат-бот дозаполняет анкету/тест из переписки (01.06.2026)

Идея Юрия: не гонять кандидата по формам, а **собрать ответы из чата**.
Сканируем вопросы вакансии (демо-анкета + тест), читаем что кандидат УЖЕ написал
в hh-чате, заполняем известное, **недостающее уточняем сообщениями в чат**, при
неясности — переспрашиваем («правильно понял, что …?»), ответы заносим в карточку.
На будущее — мессенджеры (Telegram сейчас готовить, Max позже).

Это БОЛЬШАЯ фича с продуктовыми развилками → сначала план, строить по фазам.

## Что уже есть (переиспользуем, не пишем с нуля)

- **Предквалификация** — `lib/prequalification/{start,process-answer,finalize}.ts`
  + `lib/ai-screen-prequalification.ts`. Уже: задаёт вопросы в hh-чат, ловит
  ответы, гонит через AI-вердикт (passed/failed/unclear + reasoning), хранит в
  `candidates`/qualification. ← БЛИЖАЙШИЙ каркас для «спросить → распарсить».
- **AI чат-бот** — `lib/ai/chatbot-processor.ts` (Group 22): 4-уровневая
  архитектура (pre-filter/executor/post-filter/watcher), песочница, тайминги.
  Умеет вести диалог в hh-чате и слать ответы.
- **Чтение входящих** — `lib/hh/scan-incoming.ts` (cron) — забирает новые
  сообщения кандидата из hh.
- **Структура вопросов** — демо/тест: `demos.lessonsJson` (task-блоки,
  `collectTaskQuestions`); анкета-форма: `FormFieldKey`
  (firstName/lastName/email/phone/telegram/birthDate/city) + `formFieldsConfig`.
- **Поля кандидата под заполнение** — `candidates`: city, salaryMin/Max,
  birthDate, experienceYears, workFormat, relocationReady, businessTripsReady,
  citizenship(stop-factors), languages, keySkills, industry, anketaAnswers (JSON).
- **Мессенджеры** — фундамент Telegram: `companies.telegramBotToken/…`,
  `candidates.telegramChatId/telegramUsername`, `source='telegram'`, миграция
  0158 (`telegram_channel_foundation`). Max — позже, по тому же интерфейсу канала.

## Целевой поток (per кандидат)

1. **Собрать «спецификацию полей»** — что вообще надо знать по этой вакансии:
   - обязательные поля анкеты-формы (formFieldsConfig.required);
   - вопросы демо-анкеты и теста (`collectTaskQuestions`), помеченные «важные»;
   - стоп-факторы вакансии (city/format/age/experience/citizenship/salary) —
     именно их полезно подтвердить заранее.
2. **Извлечь, что уже известно** (без вопросов кандидату):
   - из резюме hh (`hhRawData`) и уже распарсенных полей кандидата;
   - из **переписки** (history hh-чата) — AI-экстрактор читает все реплики
     кандидата и достаёт факты: «зовут X», «опыт N лет», «готов к переезду»,
     «зарплата от …», город, гражданство, ответы на конкретные вопросы.
   - Заполняем `anketaAnswers`/поля ТОЛЬКО при высокой уверенности.
3. **Определить пробелы** — поля из (1), которых нет в (2) или уверенность низкая.
4. **Спросить недостающее в чате** — по одному-два вопроса за сообщение,
   человеческим тоном («Добрый день! Уточните, пожалуйста, …»). Реестр заданных
   вопросов, чтобы не дублировать. (Каркас — предквалификация.)
5. **Поймать ответ** (scan-incoming) → AI-парсинг → занести в поле. При
   неоднозначности — **переспрос-подтверждение**: «Правильно понял, что у вас
   N лет в B2B и готовы к переезду?» → «да» → заносим; «нет/правки» → уточняем.
6. **Финализация** — когда ключевые поля собраны: отметить «анкета собрана из
   чата», прокинуть в скоринг/воронку (как сейчас делает анкета/предквал).

## AI-экстракция (контракт)

Один extractor-вызов на кандидата (или инкрементально на новое сообщение):
вход — список нужных полей/вопросов + история чата + резюме; выход (structured):
```
{ fields: [{ key, value, confidence: 0..1, sourceQuote }],
  missing: [key...],
  ambiguous: [{ key, candidateValue, clarifyQuestion }] }
```
- confidence ≥ HIGH → занести молча.
- ambiguous / средняя → задать clarifyQuestion (переспрос-подтверждение).
- missing → задать прямой вопрос.
Промпт строго инструктируем «не выдумывать»: если в чате/резюме нет — `missing`,
не угадывать. (Тот же принцип, что pre/post-filter в Group 22.)

## Безопасность / осторожность (Юрий подчёркивал)

- **Не вносить неуверенное молча** — порог confidence + обязательный переспрос
  при неоднозначности.
- **Анти-дубли вопросов** — реестр заданных (как в дожиме: дедуп по тексту/полю).
- **Не спамить** — не более N вопросов/день, группировать; уважать рабочее окно
  (`canSendNow`) и существующие rate-limit'ы.
- **Уважать kill-switch'и** AI-бота (per-vacancy/per-company/platform — Group 22).
- **Лог и обратимость** — сохранять sourceQuote и «кто заполнил: chat-autofill»,
  чтобы HR видел происхождение и мог откатить (как abuse-history undo).
- **Стоп-слова / pre-filter** на входящих (переиспользовать Group 22 pre-filter).

## Фазы (предложение)

- **Ф0 (готово/частично):** предквалификация уже умеет спросить+распарсить —
  это база. Песочница AI-бота — для теста без hh.
- **Ф1 — экстрактор из чата (read-only):** прочитать историю + резюме, заполнить
  карточку известным, показать HR «что распознали» (без отправки сообщений).
  Низкий риск, быстрый эффект. Включать feature-flag на вакансию.
- **Ф2 — проактивные вопросы недостающего** в hh-чат + ловля ответов
  (на каркасе предквалификации) + переспрос-подтверждение.
- **Ф3 — финализация → скоринг/воронка**; метрика «собрано из чата».
- **Ф4 — каналы**: вынести «отправить/принять сообщение» в интерфейс канала
  (hh | telegram | max). Telegram-фундамент есть (0158) → подключить Telegram;
  Max — позже тем же интерфейсом.

## Открытые вопросы к Юрию (решить до Ф2)

1. Где живёт «спецификация полей»: брать обязательные поля анкеты-формы + важные
   вопросы теста, или HR явно отмечает «эти поля собирать из чата»?
2. Порог уверенности для «молча занести» vs «переспросить» — дефолт, настройка?
3. Лимит вопросов в чат (в день / всего), тон/шаблоны — редактируемые на вакансии?
4. Взаимодействие с AI-чат-ботом Group 22: это его новый «навык» (он и так ведёт
   диалог) или отдельный модуль? Скорее — навык внутри chatbot-processor.
5. Что считать «анкета собрана» (минимальный набор полей) → триггер скоринга.
