### Что построено за сессию (всё на проде, кроме невыкаченного — см. §2)
Мини-фича рассылки теста + редактор:
- **Рассылка теста**: «Кандидаты → выделить → Отправить тест». Идёт через очередь
  `follow_up_messages` (branch=`test_invite`), шлёт cron `/api/cron/follow-up` (в crontab,
  раз в минуту) в hh-чат, с троттлингом. Не мгновенно. `lib/messaging/test-invite.ts`,
  эндпоинт `app/api/modules/hr/vacancies/[id]/send-test/route.ts`.
- **Окно «Отправить тест»**: показывает/редактирует текст приглашения (плейсхолдеры
  `{{name}}/{{vacancy}}/{{test_link}}`); правка сохраняется как шаблон вакансии
  (`postDemoSettings.testInviteMessage`). GET/POST в send-test.
- **`{{test_link}}`** — персональная ссылка `/test/{token}` (рендерер + cron follow-up).
- **AI оценивает ВСЕ ответы** (текст + выбор), не только текст. Выборные: зелёные ✓
  («подходящие варианты») + per-question «ИИ-проверка» уходят в AI-промпт.
- **Стадия `test_task_sent`** ставится при успешной постановке в очередь → бейдж
  «Тест отправлен». Кандидаты без hh-чата отсеиваются (тост «N без hh-чата»).
- **Дожим останавливается** на тест-стадиях (добавлены в ADVANCED_STAGES, should-stop.ts).
- **Страница теста `/test/[token]`**: убрана системная обёртка (показывается только
  контент HR), `{{Имя}}` подставляется, последний урок = экран «Спасибо» после отправки,
  редактируемая кнопка отправки (блок «Кнопка»), вариант «Другое» → поле ввода,
  предпросмотр (?as=hr) не залипает на «Спасибо».
- **Редактор теста (notion-editor)**: ИИ-проверка под вариантами; перемещение вариантов
  ↑↓; явная пометка «Другое (поле ввода)» + текст-подсказка; кнопка «Куда ведёт»
  (Следующая страница / Ссылка); выравнивание кнопки (слева/центр/справа); предпросмотр
  в табах «Тест» и «Демонстрация».
- **Прочее**: B1 — дублирование вакансии в списке копирует все настройки (`cc2cc88`);
  hh-import lock сокращён ~100с→~4с + лог в cron_runs (`a63e000`); paused-вакансия
