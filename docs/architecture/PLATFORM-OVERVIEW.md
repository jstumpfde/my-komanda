## 1. Что это
**Company24.pro = «AI Business OS»** — платформа автоматизации бизнес-процессов с AI.
Флагман — **AI-рекрутер (HR/найм)**: сквозной автопилот от отклика на hh.ru до оффера.
Вокруг — модули продаж, маркетинга, обучения, задач и др. Модель: **модульный монолит**,
продаём по модулям, объединяем бесплатно.

## 2. Стек и инфраструктура
- Next.js 16 (App Router), TypeScript, Tailwind + shadcn/ui, pnpm.
- PostgreSQL 16, Drizzle ORM (схема — `lib/db/schema.ts`, миграции — `drizzle/NNNN_*.sql`).
- AI: Anthropic (Opus 4.8 / Sonnet 4.6 / Haiku 4.5) через прокси `claude-proxy.jstumpfde.workers.dev`.
- Прод: 5.42.125.91, `/var/www/my-komanda`, PM2 `my-komanda:3000`, ветка **main**.
- Стейджинг: `/var/www/my-komanda-new-staging`, PM2 `:3001`, ветка **develop**.
- Деплой: `/root/deploy-prod-safe.sh` (дамп БД, build, pm2 reload, health-check, авто-откат).

## 4. Сквозной поток найма (AI-рекрутер)
```
1. Вакансия на hh.ru (анкета + AI-профиль/Портрет)
2. Отклик → cron hh-import (~2 мин) → candidates (stage=new)
3. Стоп-факторы (hard-фильтр) → совпало = отложенный отказ (cron, рабочее время)
4. AI-скоринг резюме (resume_score, Haiku) — РЕШАЕТ судьбу:
   <lower → мягкий отказ · [lower,upper) → демо/уточнение · ≥upper → авто-приглашение
5. Первое сообщение + ссылка на демо (hh-чат)
6. AI чат-бот ведёт диалог (injection/abuse фильтры → Executor → post-filter → watcher)
7. Демо → Анкета(ы) → Тест (стадии)
8. AI-скоринг анкеты (ai_score, Sonnet) — справочно
9. Интервью (календарь, напоминания) → Оффер/Отказ → Адаптация
Параллельно: дожим молчащим (cron ~1 мин).
```
Воронка-v2: те же шаги как настраиваемые стадии-действия (демо/предкв/тест/интервью/оффер),
рантайм за флагом. Шаблоны ролей (sales-b2b, marketer) применяются к вакансии в 1 клик.

## 5. Три системы скоринга (важно не путать)
| Балл | Когда | Движок | Роль |
|---|---|---|---|
| resume_score | до демо (боевой) | Haiku | резюме vs Портрет — **двигает воронку** |
| ai_score | после демо | Sonnet | резюме+анкета — справочно |
| ai_score_v2 («AI-Портрет») | по Портрету, по запросу | Sonnet 2 прохода | оценка по критериям Портрета (колонка AI-Порт) |
| rubric_score | shadow | Sonnet | A/B, на воронку не влияет |

## 8. Пути УЛУЧШЕНИЯ (не только упрощать)
Конкретные точки роста ценности:
1. **Прозрачный единый балл + «почему»** — кандидат/HR видит, по каким критериям прошёл/нет; калибровать пороги по факту найма (кого реально наняли → подтянуть порог).
2. **Конверсия по этапам** — на дашборде воронки показывать, где кандидаты отваливаются, и авто-подсказку «здесь теряешь N% — причина X».
3. **Больше управляемой автономности** — авто-приглашение сильных, авто-планирование интервью по календарю, авто-дожим — всё под тумблерами с дефолтом «безопасно».
4. **Библиотека готовых ролей** — расширять шаблоны (РОП, РП, маркетолог…), чтобы новая вакансия настраивалась в 1 клик с хорошим Портретом и текстами.
5. **Переиспользование кандидатов** — резерв между вакансиями: подходящего из одной вакансии предлагать в другую.
6. **Стоимость/скорость найма** — метрики «дней до найма», «стоимость нанятого», источники качества.
7. **Качество AI-диалога** — дашборд «что бот написал», быстрый разбор спорных, обучение на правках HR.
8. **Меньше кликов** — умные дефолты, единый центр настроек, генерация (Портрет/демо/тексты) из брифа.
