# Воронка v2 — спецификация

> Источник правды по редизайну воронки найма. Черновик для согласования с Юрием.
> Текущая воронка на проде НЕ трогается, пока не построим и не обкатаем v2 на стейджинге.
> Дата: 24.06.2026.

---

## 0. Проблема (зачем редизайн)

Сейчас воронка — это **23 плоских блока-тумблера**, и в них всё перемешано:

- В блоке «AI чат-бот» спрятана почти **целая вторая воронка** (триггеры, безопасность, тайминги, автономность, шаблоны отказов…).
- Одно и то же **раскидано и дублируется**: дожим — отдельными блоками; статусы hh — где-то ещё; сообщения — в каждом блоке свои; скоринг резюме — и в «Портрете», и в воронке.
- Реальный путь кандидата **ветвится** (демо ИЛИ тест ИЛИ вопрос; интервью телефон/Zoom/офис), а плоский список тумблеров это не выражает.

Итог: HR путается «где что настраивать».

---

## 1. Главная идея — три слоя

Вместо «23 блока» — **три слоя**, у каждого один дом:

| Слой | Отвечает на вопрос | Где живёт |
|------|--------------------|-----------|
| **Портрет** | Кого берём + как оцениваем резюме | Свой редактор (вкладка «Портрет») = **стадия 1** воронки |
| **Стадии** | Какой путь проходит кандидат | Вкладка «Воронка» — последовательность стадий |
| **AI-агент** | Кто общается и ведёт кандидата | Один сквозной исполнитель + база знаний + стоп-слова |

Плюс **библиотека пресетов сообщений** — шаблоны (приветствие, отказ, благодарность, дожим, напоминания), к которым обращаются стадии.

**Ключевой сдвиг мышления:** воронка = **последовательность стадий**, и всё, что раньше было отдельными блоками (дожим, статусы, сообщения, напоминания), — это **свойства стадии**, а не отдельные сущности.

---

## 2. Портрет = стадия 1

Портрет и первая стадия «Отклик → скан резюме» — **одно и то же**: Портрет задаёт критерии/пороги, на отклике они применяются → прошёл / отказ.

- **Полный редактор** Портрета остаётся (критерии «Подходит/Не подходит», пороги, идеальный профиль, стоп-факторы, авто-действия) — потому что Портрет ещё и «кого ищем вообще» (резерв талантов, аналитика, контекст вакансии).
- **В воронке** Портрет показывается стадией 1 — компактной **read-only сводкой** (пороги `<40 / 40–74 / ≥75`, что включено, стоп-факторы) + кнопка «Открыть Портрет».
- Один источник правды, два вида: полный редактор + карточка-стадия.

Логика вкладок выстраивается сама:
- **Вакансия** — что видит кандидат (описание).
- **Портрет** — кто проходит (= стадия 1).
- **Воронка** — путь: стадия 1 (Портрет, read-only) → стадии 2…N.
- **Контент** — «начинка» стадий (демо, тесты, материалы), к которой стадии обращаются.

### Поведение стадии по умолчанию — разрешающее
- **Пусто = 100% проходят.** Если в стадии 1 (Портрете) ничего не настроено — дальше идут все.
- Ограничиваешь только тем, что настроил: напр. один стоп-фактор «не берём из других городов» → проходят все, кроме иногородних.

### Где живёт «отказ / авто-приглашение»
- **Внутри стадии (решение):** включён ли авто-отказ слабых / авто-приглашение сильных, пороги, куда зовём. Это **правило прохода** стадии — у каждой стадии своё (стадия 1 — по баллу резюме; стадия 3 — по баллу теста и т.д.).
- **Снаружи (общие дефолты / библиотека):** переиспользуемое — задержка отказа (60 мин), шаблоны отказа/благодарности, пресеты дожима. Стадии наследуют, можно переопределить на стадии.
- Логика не дублируется: решение — в стадии, общие куски — в библиотеке.

---

## 3. Анатомия стадии

Каждая стадия — **самодостаточная карточка** со свойствами:

| Свойство | Что это |
|----------|---------|
| **Действие** | Что делает кандидат: скан резюме / сообщение / демо / тест / вопрос / интервью / оффер / СБ-проверка / реф-чек … |
| **Варианты** | Внутри стадии выбор: демо ИЛИ тест ИЛИ вопрос; интервью телефон/Zoom/офис; и т.п. |
| **Сообщение** | Пресет из библиотеки; учитывает день/ночь (ночью — авто-ответ «отвечу в рабочее время, а пока…»). |
| **Правило прохода** | Прошёл → следующая стадия; не прошёл / «нет, неинтересно» → благодарим + отказ. |
| **Дожим** | Пресет (мягкий / стандарт / сильный, N касаний), если кандидат молчит. На КАЖДОЙ стадии. |
| **Задержка отказа** | Не сразу; дефолт 60 мин (время передумать). |
| **hh-статус** | Автоматически меняется при входе в стадию (отклик → «первичный контакт», интервью → …, принят → «занят»). |
| **Напоминания** | Для встреч: за сутки, утром, «будете / перенесём». |

**Расширяемость:** добавил стадию (Zoom / тест / СБ / реф-чек) — пресет сообщений и дожима подтянулись автоматически.

## 4. Полная воронка (журней)

Пример сквозного пути (вариативность настраивается на каждой стадии):

### Стадия 1 — Отклик → скан резюме (= Портрет)
- AI оценивает резюме по критериям Портрета.
- **Не прошёл** → отказ (шаблон, с задержкой) + статус hh «отказ».
- **Прошёл** → дальше + статус hh «первичный контакт» + первое сообщение.

### Стадия 2 — Приветствие + первый шаг
- «Добрый день, ваша кандидатура заинтересовала, предлагаем…» + **вариант на выбор**:
  - (а) уточняющий вопрос (предквалификация),
  - (б) посмотреть демонстрацию,
  - (в) ответить на тест-вопросы.
- Учёт времени: ночью — авто-ответ + «пока можете пройти…»; днём — без ночного сообщения.
- Дожим, если молчит. «Нет, неинтересно» → благодарим (шаблон) → отказ «сам отказался» + статус hh.

### Стадии 3…K — Контент-этапы (повторяемые)
- На каждой: демо / тест / вопросы / тест-задание (настраиваемо).
- Прошёл по правилу → следующая стадия; не прошёл → благодарим + отказ (задержка 60 мин).
- Несколько этапов подряд: «посмотрел демо + ответил» → следующий пул вопросов / тест-задание и т.д.

### Стадия L — Интервью (тип настраивается)
- **Телефон**: HR звонит ИЛИ бот-звонилка задаёт вопросы и записывает ответы.
- **Zoom** / **Офис**.
- **Согласование времени** — два варианта (решить, оставлять ли оба):
  - бот согласует время в чате и пишет в календарь, ИЛИ
  - кандидату присылается ссылка для самозаписи.
- **Напоминания**: за сутки, утром, уточнение «будете / перенесём».
- Цепочка интервью расширяема: телефон → Zoom → офис → доп. собес / тест.

### Стадия M — Оффер
- Генерация оффера + подпись → ждём подтверждения (дожим) → согласование даты выхода.

### Стадия N — Проверки (СБ + реф-чек)
- Проверки по сервисам (ИНН, банкротство, реестры…).
- Реф-чек: берём у кандидата номер прошлого работодателя → письмо/звонок с просьбой об отзыве.

### Финал — Принят
- Статус hh/система «занят», карточка сохраняется.
- Переход в отдельный модуль **Адаптация** (онбординг: обучение + тесты).

## 5. AI-агент — сквозной исполнитель

Бот — **один агент на кандидата**, который **читает воронку**, а не хранит её копию.

### 4 слоя контекста (что бот читает)
1. **Стадии воронки** — карта стадий + что дальше.
2. **Состояние кандидата** — где он, история, балл, назначенные встречи.
3. **Вакансия + база знаний (FAQ)** — зарплата, график, релокация, про компанию + любые добавленные темы.
4. **Стоп-слова + безопасность** — injection / мат / abuse.

### 3 действия (что бот делает)
1. **Отвечает кандидату** — по теме / из базы знаний («когда интервью», «что за тест», «какая зарплата», «можно удалёнка»).
2. **Ведёт по воронке** — шлёт сообщения стадий, делает дожим, согласует встречи, двигает дальше (в рамках разрешённой автономности).
3. **Эскалация HR** — вне темы / стоп-слово / неуверен.

### Режимы — один агент
- **Реактивный** (кандидат спросил) и **проактивный** (событие воронки) — это один и тот же бот с одним контекстом; разница лишь в триггере.

### Настройка бота = только поведение
Тон, на какие темы отвечает, **база знаний (FAQ)**, безопасность (стоп-слова, injection/мат/abuse), тайминги (задержка, короткое «минутку…»), **автономность** (что ему можно делать в воронке), эскалация в Telegram, AI-наблюдатель.
**Контент и логику стадий бот НЕ дублирует** — берёт из стадий. Так раздел бота перестаёт быть «второй воронкой».

### Границы
- «Интервью назначает только HR» (если так настроено) — бот лишь согласует время.
- Настраивается: рассказывает ли бот про будущие стадии.
- Автономность включается по одной возможности, с наблюдением за диалогами.

## 6. Сквозные механики (свойства стадий / общие библиотеки)

- **Дожим** — пресеты (мягкий/стандарт/сильный, N касаний). Свойство каждой стадии, не отдельный блок.
- **Статусы hh** — mapping «стадия → статус hh», автосмена при входе в стадию (dual-write).
- **Напоминания** — для стадий-встреч.
- **Пресеты сообщений** — библиотека шаблонов (приветствие, отказ, благодарность, дожим, напоминания) в одном месте; стадии ссылаются.
- **Тайминги день/ночь**, **задержка отказа** (дефолт 60 мин).

## 7. Что это консолидирует (из текущих 23 блоков)

| Было (плоские блоки) | Стало |
|----------------------|-------|
| AI-скоринг резюме + Стоп-факторы по резюме | **Стадия 1 (Портрет)** — read-only сводка в воронке |
| Первое сообщение, Аварийное, Дожим, Дожим по тесту, Автоответ с тест.заданием | **Свойства стадий** (сообщение + дожим) + библиотека пресетов |
| Стоп-слова в чате, FAQ-шаблоны, Хочет созвониться | **Слои AI-агента** (стоп-слова, база знаний, эскалация) |
| AI чат-бот (с целой воронкой внутри) | **AI-агент = поведение + база знаний**, читает стадии |
| Предквалификация, Демо, Тест, Тест-задание, Видео-визитка, Анкета, AI-скрининг анкеты | **Стадии-контент** (тип + вариант + правило) |
| Интервью | **Стадия «Интервью»** с вариантами (тел/Zoom/офис, согласование, напоминания) |
| Реф-чек, Оффер | **Стадии** (+ новая стадия «СБ-проверка») |
| Стадии воронки (канбан + hh) | **hh-статус = свойство стадии** |

## 8. Решения (согласовано с Юрием, 24.06)

1. **Согласование интервью** — оставляем **оба** варианта (бот согласует время в чате И ссылка на самозапись); HR выбирает на стадии.
2. **Бот-звонилка** — **позже**. На первом этапе телефонное интервью = HR звонит сам.
3. **СБ-проверки** — через **Dadata** (для начала).
4. **Адаптация** — **позже**, отдельным модулем. Последняя стадия воронки = **«Нанят»**; на ней воронка заканчивается.
5. **Ветвление** — **да**, если не усложнит сильно: условные переходы (балл/ответ → стадия A или B) помимо линейного «прошёл/не прошёл».
6. **Строим v2 рядом** с текущей воронкой — прод не трогаем. ✅

## 9. План реализации (фазами, через стейджинг)

Текущая воронка на проде работает — её не трогаем, v2 строим рядом.

- **Фаза 0** — этот спек + мокап карточки стадии + стадии «Портрет (read-only)» и «Интервью». Согласовать.
- **Фаза 1** — каркас: «стадии + свойства» (действие, сообщение-пресет, правило, дожим, hh-статус), Портрет как стадия 1 (сводка).
- **Фаза 2** — стадии-контент (демо/тест/задание/вопрос) с вариантами.
- **Фаза 3** — AI-агент v2: 4 слоя контекста + база знаний + автономность поверх стадий.
- **Фаза 4** — интервью (тип + согласование + напоминания), оффер.
- **Фаза 5** — СБ/реф-чек, переход в Адаптацию.

Каждая фаза — отдельная ветка → стейджинг → прод, с надзором (predeploy-guard).
