# Эталон Юрия — «что нужно» + Definition of Done

> ЧИТАТЬ ПЕРВЫМ перед любой задачей. Этот файл — источник правды по тому, ЧТО хочет
> Юрий и что значит «сделано». Если Юрий что-то поправил — правка СРАЗУ сюда, чтобы
> не объяснять второй раз. Любой агент (надзорный/исполнитель) сверяется с этим файлом.

## 1. Кто Юрий и что он ценит
- Основатель Company24.pro, маркетолог с 30+ лет опыта. Думает продуктом и продажами.
- Ценит: **скорость > перфекционизм, НО не ценой поломок**; сделано целиком, а не кусками;
  совпадение с согласованным (макеты, договорённости), а не «похоже»; продающий смысл
  (тексты/воронка/конверсия), а не сухая механика.
- Не выносит: переобъяснять одно и то же; «сделал и встал»; регрессы у соседних функций;
  простой/хождение по кругу.

## 2. Definition of Done (задача НЕ закрыта, пока не выполнено ВСЁ)
1. Сделано **целиком по договорённости/макету** — не половина, не «черновик».
2. `pnpm build` (Node 20) зелёный + `npx tsc --noEmit` чисто по тронутым файлам
   (Turbopack НЕ тайпчекает → tsc обязателен; иначе рантайм-404/ReferenceError).
3. **Надзорное адверс-ревью** перед прод-деплоем (см. §5) — 0 блокеров/мажоров.
4. Деплой **стейджинг(canary)→прод** через `/root/deploy-prod-safe.sh`; health 200,
   рестарты не растут, BUILD_ID есть.
5. **Показать готовый результат** (что и где проверить), без хеджа. Не спрашивать
   «дальше?» если объём согласован — брать следующий кусок.

## 3. Инварианты «НЕ ЛОМАТЬ» (проверять на каждом изменении)
- **Legacy/существующее — байт-в-байт.** Флаги default-OFF (portrait_scoring и т.п.):
  старые вакансии и Орлинк работают как раньше. Новое — только за флагом.
- **Ничего не терять при импорте/переносе.** hh-импорт сохраняет ВСЕ поля
  (вступление/about, обязанности, требования, условия, частота выплат, язык, график,
  категория). Если поле есть в источнике — оно должно дойти.
- **Никакого мёртвого UI.** Кнопка/селект/поле либо реально работает, либо его нет.
- **Актуальные model id — только через реестр `lib/ai/models.ts`** (AI_MODEL_MAIN =
  `claude-sonnet-5`, AI_MODEL_FAST = `claude-haiku-4-5-20251001`; Opus точечно —
  `claude-opus-4-8`). НЕ хардкодить id в коде. Sonnet 5 НЕ принимает temperature/top_p.
  Грепать кодбазу на устаревшие при любом AI-касании [[ai-model-deprecation-sonnet4-to-46]].
- **Деньги/право чувствительно:** отказы нейтральны; ОВЗ/возраст/гражданство не ставить
  ложно; авто-отказ по защищённым признакам — осознанно оставлен, не выпиливать.
- **Общий worktree:** `git add` по именам, не `-A` [[shared-worktree-never-git-add-all]].

## 4. Как понимать «что нужно» (правила интерпретации)
- Если есть согласованный **макет/договорённость** — восстановить его САМ из транскрипта
  и сделать ЦЕЛИКОМ. Не упрощать до косметики поверх старого скелета.
- На развилке, где у меня есть рекомендация, — взять её и сделать, не гейтить вопросом.
  Вопрос только на настоящей развилке (необратимое/реальный продуктовый выбор).
- При молчании ~5 мин — идти по очереди задач самому [[autonomous-queue-on-silence]].
- Каждое касание фичи: проверить всю цепочку UI→API→данные e2e, а не один файл.

## 5. Надзор перед деплоем (обязательный ритуал)
Перед КАЖДЫМ прод-деплоем нетривиального изменения — адверс-ревью (workflow
`predeploy-guard`, 3 линзы + верификация), который проверяет diff против:
1. §2 Definition of Done (сделано целиком? build/tsc?).
2. §3 Инварианты (legacy байт-в-байт? ничего не потеряно? нет мёртвого UI? model id?).
3. Совпадение с согласованным макетом/договорённостью (§7).
Находки major/blocker — чиню ДО деплоя. Минор — фикшу или явно фиксирую как принятое.
Деплой только после 0 блокеров/мажоров + стейджинг-canary зелёный.

## 6. Закрепление (чтобы не объяснять дважды)
Любая поправка/недовольство Юрия → СРАЗУ дописать сюда (или в [[MEMORY]]) конкретным
правилом. Цель: каждое «я же говорил» превращается в строку этого файла навсегда.
