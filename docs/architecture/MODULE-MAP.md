## Вакансия — табы (редактор)
URL: /hr/vacancies/[id]?tab=...
- **Кандидаты** (tab=candidates) — список кандидатов (рабочий, 1505 у маркетинговой)
- **Аналитика** (tab=analytics) — воронка/источники/распределение скора (работает)
- **Анкета** (tab=anketa) → переименовать в «Вакансия» (D18). Блоки 1-9 + AI-профиль
  кандидата + правая AI-панель (vacancy-advisor.tsx)
- **Демонстрация** (tab=demo) — конструктор уроков; в конце системная анкета
- **Тест** (tab=test) — конструктор теста (квиз + скоринг score-test-objective.ts)
- **Исходящий подбор** (tab=outbound)
- **Настройки** (tab=settings) — под-табы (см. ниже)

⚠️ Deep-link `?tab=settings/anketa/analytics` на АКТИВНОЙ вакансии редиректит на
«Кандидаты» (намеренный гард P0-50, page.tsx:590) → Q1.

## Вакансия → Настройки — под-табы (section=...)
Брендинг · Источники · Воронка · **Конструктор воронки [Beta]** · Сообщения · Дожим ·
AI чат-бот · Расписание · Интеграции.
- Это per-vacancy ЗЕРКАЛО дефолтов компании. Воронка фрагментирована по этим под-табам
  (см. BIG-PICTURE §A, центральный узел).
- Целевое: всё свести в «Конструктор воронки» (D1).

## Дефолты компании (Настройки найма) — табы
- **Основные**: аварийное отключение AI, выбор компании в анкете, описание компании,
  авто-сбор обратной связи (→ в вакансию, D9), хранение ПДн (ФЗ-152), webhooks, Битрикс24
- **Интеграции**: hh.ru подключено + площадки (Авито/SuperJob/Яндекс — «Скоро»)
- + (на уровне вакансии видели) Расписание/Воронка/Сообщения/Стоп-факторы
