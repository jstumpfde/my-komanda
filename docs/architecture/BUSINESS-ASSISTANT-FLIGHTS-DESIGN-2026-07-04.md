# Модуль «Бизнес-ассистент» — дизайн, релиз 1: Авиабилеты

Дата: 2026-07-04. Статус: одобрено Юрием, готово к плану реализации.

## Идея модуля

«Бизнес-ассистент» — новый модуль my-komanda: поиск выгодных предложений
(билеты, отели, промокоды, рестораны/бани, в перспективе — голосовой
бот-консьерж). Каждая категория — отдельный подпроект со своим спеком.
Общая идея: не пытаться строить собственный поисковик цен по рынку (нужен
доступ к GDS, месяцы разработки), а собирать готовые аффилиатные источники
+ курировать "слив"-каналы, там где агрегаторов не хватает.

Порядок подпроектов (по интересу Юрия, 04.07.2026):
1. **Авиабилеты** — этот релиз.
2. **Отели / путёвки** — тот же паттерн (аффилиат-виджеты Ostrovok/Level.Travel).
3. **Промокоды** — активный «радар» по промокодам на разные категории
   (не просто лента, а поиск действующих кодов). Юрий не знает, где брать
   источники — нужно исследовать при подготовке спека.
4. **Рестораны и бани** — бронирование + обязательно применение промокода/
   скидки (пример: баня по промокоду может стоить на 45% дешевле обычной
   цены, а не 15%, которые есть сейчас без сервиса). Вероятно переиспользует
   «радар» промокодов из п.3, а не отдельный движок.
5. Дальше — концерты/мероприятия, столики (детали спеком позже).
6. Голосовой бот-консьерж (звонит, узнаёт, записывает, напр. к врачу) —
   отдельный крупный проект. **Важно**: модуль `dialer` (AI-агент звонков) в
   кодовой базе сейчас только моки, реальной телефонии нет — звонилку для
   бота придётся строить с нуля, ничего готового не переиспользовать.

## Видимость модуля (ВАЖНО — решение 04.07)

Модуль **не включается по умолчанию всем клиентам**, пока не готов. В
`lib/auth.tsx:173-204` видимость модулей по ролям зашита жёстко: сейчас все
клиентские роли (director, hr_lead и т.п.) видят только `hr`. Для
`business_assistant`:
- НЕ добавляем в дефолтный список ролей клиента.
- Включаем только через `companies.enabled_modules` — точечно, для тестовой
  компании Юрия, пока не готово показывать всем.
- Когда Юрий скажет «готово, включай всем» — добавить `business_assistant` в
  список ролей клиента в `getVisibleSections()` (по образцу `hr`), одной
  правкой откроется всем компаниям сразу.

## Релиз 1 — «Авиабилеты»

### Место в навигации
- Новый модуль `business_assistant` в `lib/modules/types.ts` (ModuleId) и
  `lib/modules/registry.ts`, basePath `/business-assistant`.
- Первый пункт меню — «Авиабилеты» (`/business-assistant/flights`).
- Остальные будущие пункты (Отели, Промокоды, Рестораны/бани) — показать
  как пункты со значком «Скоро» (паттерн уже есть в `dialer/settings`), чтобы
  было видно направление развития модуля.

### Страница «Авиабилеты»
- Форма поиска: откуда/куда (с автокомплитом), даты, пассажиры, класс —
  свои shadcn/ui компоненты (без чужого iframe-виджета — не соответствовало
  бы стилю платформы).
- Результаты — два блока:
  1. **Обычные рейсы** — Travelpayouts Flight Data API (цены по направлению).
  2. **Составные маршруты** — Kiwi Tequila API (умеет сама склеивать билеты
     разных перевозчиков в один маршрут — механика «билет дальше, но
     дешевле»), с бейджем размера экономии.
  - Кнопка «Купить» на каждой карточке — диплинк на Aviasales/Kiwi с нашим
    `marker`/`affid`, открывается в новой вкладке; подпись «бронирование и
    оплата — на сайте партнёра».
- Блок **«🔥 Горячие предложения»** — лента находок из Telegram-каналов.

### Бэкенд
- `app/api/modules/business-assistant/flights/search/route.ts` — сервер
  дёргает Travelpayouts Data API + Kiwi Tequila API, прячет ключи, отдаёт
  нормализованный JSON в общем формате для обоих источников.
- `app/api/modules/business-assistant/flights/deals/route.ts` — отдаёт
  сохранённую ленту `flight_deals` (пагинация).
- Новая таблица `flight_deals` (платформенная, БЕЗ `company_id` — источник
  публичный, не клиентские данные): маршрут, цена, канал-источник, ссылка на
  пост, `ai_extracted_json`, `created_at`; уникальность по ссылке на пост —
  дедуп.
- Крон `flight-deals-ingest`: переиспользует существующую Telegram-userbot
  инфраструктуру (GramJS, тот же `api_id`/`api_hash`, что в модулях
  «Telegram-постинг» / «Радар контента»). Читает список публичных каналов из
  конфига (см. ниже), забирает новые посты с прошлого запуска, Claude Haiku
  вытаскивает маршрут/цену/ссылку из текста, пишет в `flight_deals`. Опрос
  редкий (раз в 15–30 мин) — чтобы не спровоцировать бан аккаунта за
  агрессивную автоматизацию. Список каналов — только явно одобренные Юрием,
  автоматического вступления в новые каналы нет.

### Регистрация в партнёрках (делает Юрий, не код)
**1. Travelpayouts (Aviasales)**
1. Зарегистрироваться на [travelpayouts.com](https://www.travelpayouts.com/ru/)
   (email + пароль).
2. В кабинете подключить программу «Авиасейлс для бизнеса» — принять условия,
   нажать «Подключиться».
3. В разделе «Инструменты» получить персональную ссылку вида
   `https://tp.media/r?marker=...` — значение `marker=` и есть партнёрский ID
   для интеграции.
4. В разделе «Баланс» → «Реквизиты» указать способ вывода средств (нужен
   статус самозанятого/ИП/физлица — Travelpayouts платит раз в месяц).
5. Дать мне: `marker` + API-токен (выдаётся в личном кабинете, раздел API).

Источники: [Travelpayouts — партнёрская программа Aviasales](https://www.travelpayouts.com/ru/offers/aviasales-partnerskaya-programma/), [пошаговая инструкция Travelpayouts](https://www.travelpayouts.com/ru/blog/poshagovaya-instrukciya/)

**2. Kiwi.com Tequila API**
1. Зарегистрироваться на [tequila.kiwi.com](https://tequila.kiwi.com/) —
   бесплатно.
2. После входа перейти в раздел «My applications» → «+ Add application».
3. Выбрать тип партнёрства: **Kiwi.com Affiliate Program** (нужен именно
   этот, а не «Book with Kiwi.com» — он для полноценного букинга с обработкой
   оплаты).
4. Выбрать продукт: **Search** (поиск, нам этого достаточно для MVP).
5. Получить API key на странице приложения.
- ⚠️ Нюанс: полноценное одобрение (production-доступ, не sandbox) у Kiwi
  зависит от типа партнёрства и ожидаемого объёма — возможен ручной review
  с их стороны. На старте будем работать в sandbox/тестовом ключе, если
  production не одобрят сразу.

Источники: [Kiwi Tequila API key guide](https://kiwicom.github.io/margarita/docs/guide-tequila-api-key), [Kiwi.com affiliate program API — Travelpayouts](https://support.travelpayouts.com/hc/en-us/articles/360019237899-Kiwi-com-affiliate-program-API)

**Пока ключей нет** — деплою модуль на стейджинге с мок-данными (реалистичные
фейковые результаты поиска), включу реальные вызовы API, как только Юрий
пришлёт `marker`/токены.

### Риски, зафиксированные явно
1. Без креплённых партнёрских ключей — модуль не монетизируется, только UI.
2. Мониторинг Telegram через личный аккаунт Юрия — риск ограничений при
   слишком частом опросе; опрашиваем редко (15–30 мин), без автовступления
   в новые каналы.
3. Kiwi production-доступ не гарантирован сразу — может остаться в sandbox
   на старте.
4. Модуль виден только через точечный `companies.enabled_modules`, пока
   Юрий не даст добро показать всем клиентам.
