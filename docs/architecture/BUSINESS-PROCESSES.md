# Бизнес-процессы платформы — упрощение + улучшение (26.06.2026)

Подробный разбор ВСЕХ бизнес-процессов: как работает, что упростить (без потери результата),
что улучшить, где больше отдать AI-агенту, какие пресеты. Источник — 5 глубоких аудитов.
Поддерживать актуальным (еженедельный авто-аудит). Связано: [AUDIT-2026-06-26.md], [PLATFORM-OVERVIEW.md].

## 🔑 4 СКВОЗНЫХ ВЫВОДА (корень всего)
1. **Дефолты работают ПРОТИВ потока.** `autoInvite=false`, дожим `preset=off`, расписание не задано → сильные кандидаты теряются, дожим молчит. → Сменить дефолты на «безопасно-активные».
2. **Параллельные дубли везде** (4 скоринга · 2 статуса · 3 воронки · 2 дожима · 3 хранилища стоп-факторов · 6+ источников текстов). → «Одна система на задачу», legacy выпиливать.
3. **AI недо-используется** — много ручных шагов, которые агент может делать под тумблером (авто-приглашение, авто-дожим, авто-планирование интервью, объяснение решений, генерация Портрета/текстов).
4. **Нет пресетов и умных дефолтов** — каждая вакансия/компания настраивается с нуля. → Библиотека ролей/сообщений/порогов + company-defaults = старт «в 1 клик».

## 1. НАЙМ (отклик → оффер)
**Упрощения (быстрые, низкий риск):**
- `autoInvite` сильных (score≥upper) — сделать дефолтом ON с тумблером «требовать ручного утверждения» (сейчас даже сильные паркуются в «новые», AI скорит «в пустоту»). `lib/hh/process-queue.ts:326`.
- Дожим `preset="standard"` по умолчанию (сейчас `off`). `vacancy-followup-settings.tsx`.
- Дефолтное расписание вакансии ПН–ПТ 09–18 (иначе приглашения ночью).
- Один путь воронки (v2 за флагом, legacy выпилить) — убрать двойную обработку в process-queue.

**Улучшения (конверсия/скорость):**
- Видимые AI-баллы в карточке + «почему» (🟢 рекомендует / 🟡 уточнить / 🔴 нет).
- Score-driven routing: ≥80 → сразу анкета/интервью (минус неделя); 40–60 → демо; <40 → отказ.
- Badge «вы теряете кандидата» (демо открыт 3 дня назад, ответа нет) + быстрое «спасательное» сообщение.
- Batch-приглашение всех «keep_new» сильных.

**AI-агент (с тумблерами безопасности, дефолт безопасный):**
- Авто-приглашение сильных · авто-отказ слабых (лог + отложенно) · авто-предкв середины · бот сам шлёт анкету/тест (низкий риск) · авто-предложение слотов интервью (бот предлагает, HR подтверждает) · аудит-лог всех авто-действий (таблица `ai_agent_actions` — для доверия).

**Пресеты:** «Быстрый найм» (junior/масс) / «Премиум» (senior, ручной разбор) / «Аутсорс» (бот ведёт всё). Company-level дефолты (расписание, дожим, стоп-факторы) — настроил раз, наследуется.

## 2. ОЦЕНКА КАНДИДАТА (Портрет + скоринг)
**Упрощения:** свести 4 балла к ОДНОЙ прозрачной оценке в UI (resume до демо → final после; rubric/v2 — в тень/A-B). Один контур требований — Портрет (Spec), legacy `aiProcessSettings` авто-мигрировать и выпилить. Один источник стоп-факторов.
**Улучшения:** объяснимость по критериям (цитата из резюме «почему 62»); калибровка порогов по факту найма (hire_rate по бакетам → рекомендация порога); синонимы=ИЛИ (уже частично); анти-гейминг анкеты (штраф за generic, бонус за цифры/проекты); hard/soft применять везде, не только в Портрете.
**AI-агент:** авто-генерация Портрета из брифа/сайта (intake-to-spec уже есть); авто-калибровка порогов; авто-объяснение решения кандидату и HR; авто-сравнение топ-кандидатов.
**Пресеты:** библиотека Портретов по ролям (marketer/sales-b2b есть → +backend/frontend/РОП/РП/саппорт), старт в 1 клик с выверенными порогами.

## 3. КОММУНИКАЦИИ (сообщения/дожим/бот)
**Упрощения (большой выигрыш):** единый «Центр сообщений» — ОДИН набор редактируемых пресетов в UI (сейчас тексты в 6+ файлах + 4 таблицах + 9 хардкод-слотов = 19 точек синхронизации); ОДНА очередь отправки (`outbound_messages`) с адаптерами каналов (hh/tg/email); убрать риск двойной отправки (legacy-дожим + v2-дожим работают параллельно — нет гейта).
**Улучшения:** предпросмотр всей цепочки «как видит кандидат»; A/B текстов + аналитика open/reply; персонализация по тону компании.
**AI-агент:** авто-генерация всех текстов воронки из брифа/тона; адаптивный дожим (по поведению: открыл-не досмотрел → другой текст); conversational-бот вместо форм-анкеты; авто-разбор спорных диалогов.
**Пресеты:** наборы по тону/роли (есть [funnel-messages-catalog.md]), применение в 1 клик; кастомные дни касаний в UI (сейчас в коде).

## 4. РЕЗЕРВ · ИСХОДЯЩИЙ · АНАЛИТИКА
**Резерв** (см. [proposals/reserve-spec.md], строю сейчас): источник чипом (✅ сделано), добавление из отборов (✅ работает), жизненный цикл архив→корзина (5 мес, 152-ФЗ — инкремент 2), прогрев авто-касаниями (инкремент 3), отказ→резерв с отдельным сообщением. AI: не авто-матч (отклонён), а «предложить в резерв при отказе».
**Исходящий подбор** (позже): бот ищет резюме на hh по сниппетам (бесплатно) → AI-скоринг сниппета → открываем платно ТОЛЬКО ≥ порога (градус, дефолт 75) → приглашаем откликнуться. Доделать: формат hh-приглашения (negotiations — сверить с докой!), авто-режим (cron), точная проверка доступа к базе. Не связан с Резервом.
**Аналитика** (улучшить): конверсия по этапам («где теряешь N%»), стоимость/скорость найма (time-to-hire), качество источников (конверсия источник→найм), структурированные причины отказа, AI-метрики (точность скоринга vs факт найма), интеграция flight-risk.
