## Настройки найма — таб «Интервью» / «Служебное» (батч 1)

### Битрикс24 / интеграции
- Подключение Битрикса должно быть на **уровне платформы** (админ подключает),
  клиенту НЕ нужно подключать повторно. Сейчас пересекается (платформа + клиент).
- → вынести platform-level подключение в зону администратора.

### Календарь интервью («Скоро» на табе «Интервью»)
- Это НЕ просто «потом/заглушка». Это настройка того, **КАК кандидату назначается интервью**:
  - либо кандидату высылается ссылка на календарь,
  - либо ассистент **Нэнси предлагает кандидату свободные слоты** из календаря.
- Эту настройку уже можно ставить в работу (фича реальная).

### «Часы для записи кандидатов»
- Настраивается ОДИН раз на всю компанию (company-level) — это ок.
- НО сейчас рассчитано на **одного HR**. Надо продумать **мульти-HR**:
  - у каждого HR свой календарь;
  - общий слот-пул, допускающий дубли;
  - у каждого HR ещё и личный календарь;
  - назначать: какая вакансия → какой HR.

## /hr/talent-pool (Резерв) — видение и пересечения (батч 5)

  — внутренняя завязана на рефералку. У обеих: трекинг-ссылки (Источник+Название размещения),

### Блоки AI-скрининг анкеты / Контент-шаг / Автоответ (батч 34)
- **AI-скрининг анкеты** (Sheet): оценивает ОТВЕТЫ анкеты (не резюме!). Пороги ПОСЛЕ демо: Верхний 75%
  (зелёный) / Нижний 50% (жёлтый) → Отличный ≥75% (заголовок «Выберите время для встречи», типы Звонок/
  Онлайн/Офис, действие: **планировщик слотов на интервью**) / Средний 50-74% / Не подходит <50%. Режим
  Автоматический(Скоро)/Ручной. Юрий: «для оценки анкет = ответы на вопросы».
  ⚠️ **ВТОРОЙ набор порогов** (75/50) отдельно от порогов резюме (75/40 в AI-скоринг резюме). И финальный
  экран «выберите время» ДУБЛИРУЕТ финал блока Демонстрация (батч 32). Слотинг интервью — снова. Свести пороги/финалы.
- **Контент-шаг (прототип)** (Sheet): «единый вход в шаги воронки» — режимы Презентация / Демонстрация / Тест(вопросы)
  / Тестовое задание в ОДНОМ блоке (Демо/Тест = те же что в отдельных вкладках, Презентация — новый текстовый).
  Юрий: «не помню для чего, наверное выбирать какой шаг; кажется, это надо где-то в воронке делать — смотри как лучше».
  → это и есть КОНЦЕПТ консолидации контент-шагов (мой `content_step` прототип). Решить: довести как единый блок или убрать.
- **Автоответ с тестовым заданием** (Sheet): бот сам шлёт сообщение через N мин после сдачи анкеты (напр. тест-задание),
  HR не разбирает руками; «Учитывать рабочее время» (ночью → утром); текст + ссылка на тест. Юрий: «разберись —
  нужен или нет, с ним или без». → кандидат на упрощение/слияние с «Тестовое задание».

### Блоки Тестовое задание / Стоп-слова / Хочет созвониться (батч 35)
- **Тестовое задание** (Sheet): отдельная ступень задание→ответ→AI-проверка (после анкеты, до интервью).
  Текст задания, Срок (дней), Формат ответа, Проверка (AI оценивает / HR решает), Критерии для AI,
  Сообщение после теста, Дожим по тесту. Юрий: разобраться, нужен/с ним или без (как и Автоответ — пересекаются:
  Автоответ шлёт тест-задание ⟂ Тестовое задание = отдельная ступень ⟂ Дожим по тесту. Свести в один «Тест».).
- **Стоп-слова в чате** (Sheet): слова в ответе кандидата (неактуально/не подходит/отказ/…) → стадия «Отказ»,
  дожим отменяется.
- **Хочет созвониться** (Sheet): ключевые слова (созвон/позвоните/телефон/…) → один из 3 эскалационных шаблонов,
  дожим на демо. Реакция: Предложить слот+демо(Скоро) / Сразу слот(Скоро) / Настоять на демо.

**⭐⭐ КЛЮЧЕВОЙ ВЫВОД Юрия (объединить чат-сценарии):** Стоп-слова, Хочет созвониться (и FAQ, и AI-чат-бот) —
это ОДИН И ТОТ ЖЕ паттерн: **обнаружили ключевое слово/интент в чате кандидата → реакция (шаблон-ответ +
действие со стадией)**. Сейчас это РАЗНЫЕ блоки. Юрий описывает варианты: «хочет созвониться → сценарий А;
хочет про зарплату → сценарий Б; написал "отказываюсь"/"неактуально" → благодарим + переводим на другую стадию».
→ Свести в ОДНУ систему **«чат-сценарии / интенты»**: интент (созвон / зарплата / отказ / вопрос / …) →
реакция (шаблон + смена стадии + опц. дожим). Это убирает 3-4 блока и делает чат-логику единой и расширяемой.
Стоп-слова сейчас «просто стоят» (blanket автоотказ) — а должны быть частью сценариев с шаблоном ответа.
