## Вывод верификации (по официальной доке hh: github.com/hhru/api)

**Текущая реализация `inviteResumeToVacancy` ([lib/hh/outbound.ts:285](../lib/hh/outbound.ts)) НЕПОЛНАЯ.**

Сейчас код делает напрямую:
```
POST /negotiations { vacancy_id, resume_id, message }   // application/json
```

Официальная дока hh (`docs/employer_negotiations.md` в репо hhru/api) требует
**обязательный предварительный шаг**, которого в коде нет:
1. Получить список **применимых вакансий работодателя** к данному резюме — оттуда
   приходят `arguments` (обязательные параметры) и `resulting_employer_state`
   (ожидаемое состояние созданного приглашения).
2. Создать приглашение, **передав это состояние/аргументы** (а не только
   vacancy_id+resume_id+message).

Операция в hh OpenAPI называется **`invite-applicant-to-vacancy`**. Точные
путь/Content-Type/поля тела раскрыты только в OpenAPI-спеке (не в .md),
и их НАДЁЖНО подтвердит только реальный тестовый вызов под аккаунтом с активным
доступом к базе резюме.

Ограничения hh, подтверждённые докой:
- Требуется активированный платный доступ к базе резюме.
- На пару «вакансия + резюме» — только один отклик/приглашение.

## Что доделать (задача #4)

3. Сверить точный путь и Content-Type по hh OpenAPI (`invite-applicant-to-vacancy`) —

   а не JSON; это надо проверить.
4. **Обкатать на 1 реальном резюме под надзором Юрия** — это outward (приглашение
