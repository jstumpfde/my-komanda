- `components/editor/` — вся директория (9 файлов, ~1.4k строк)

- `components/vacancies/course-tab.backup.tsx` (375 строк)
- `components/knowledge/editor.tsx` (249 строк)

удалением — финальный grep по именам, чтобы убедиться что нет
ссылок в комментариях или динамических импортов.

## Рекомендации для следующей итерации

### 1. Редакторы — Юрий выбирает (приоритет)
- **«Мастер»**: `NotionEditor` (`components/vacancies/notion-editor.tsx`)
  + обёртка `MaterialEditor` (`components/core/material-editor.tsx`).
- **Удалить безопасно** (нет внешних импортов):
  - `components/editor/` целиком (9 файлов)
  - `components/vacancies/notion-course-tab.tsx`
  - `components/vacancies/course-tab.backup.tsx`
  - `components/knowledge/editor.tsx`
- **Размер удаления**: ~2.7k строк, 12 файлов. Снижение когнитивной
  нагрузки + меньше путаницы при поиске.

### 2. Структура `components/editor/` исчезает после удаления
Если оставить только NotionEditor — папка `components/editor/` пустеет
полностью. Можно удалить вместе с `add-block-button`, `block-card`,
`block-toolbar`, `section-sidebar`, `variable-picker`, `theme-colors.ts`,
`types.ts` — все они импортированы только Unified/UniversalBlockEditor'ами.

### 3. Deprecated AI-chatbot (Группа 30) — отложить
Подождать пока существующие компании не обновят JSON через UI. Удаление
полей `abuseFilter.{sensitivity,action}` без миграции данных сломает
типизацию старых записей. Лучше через 1-2 спринта добавить миграцию
«удалить устаревшие поля из jsonb» + потом убрать типы.

### 4. Console-логи
Стиль `[module] structured json` — выдержанный. Менять не нужно. При
появлении observability-стека (Sentry/Datadog) — заменить на единый
logger-helper, но это отдельный спринт.

### 5. Потенциальные оптимизации (не сейчас)
- `notion-editor.tsx` 2.7k строк в одном файле — кандидат на разбиение,
  но риск сломать инлайн-зависимости велик. Только после Group 36+
  стабилизации.
- Дубли типов между `ai-chatbot-settings.tsx` и `chatbot-processor.ts`
  (`AbuseFilterSettings`, `RejectionMessages`, `ResponseTiming`). Можно
  вынести в общий `lib/ai/chatbot-types.ts`, но это touch'ит много мест.
