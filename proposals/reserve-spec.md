# Резерв (Talent Pool) — спецификация (решения Юрия 26.06)

Цель: «вечный движок кандидатов» — никого не теряем, видно откуда каждый, прогрев авто,
законный жизненный цикл хранения.

## 1. Сбор кандидатов (с пометкой ИСТОЧНИКА — видно откуда)
Источники (структурный `sourceType`):
- **manual** — добавлен вручную.
- **vacancy_reject** — отказник вакансии (добавляется из отбора). Хранить ссылку на вакансию.
- **referral** — пришёл через реферальную ссылку (помечать «реферальный»).
- **form** — отклик с публичной формы.
- **outbound** — найден исходящим подбором (позже).
- **csv/import** — загрузка.
В Базе источник показываем чипом (иконка+цвет+подпись), чтобы картина была понятна.

## 2. Прогрев (авто) — отдельный заход
Cold → в прогреве → горячий по расписанию авто-касаний. (Реализация — Инкремент 3.)

## 3. Отказ → Резерв (вместо авто-матча на вакансии)
- Авто-матч на новой вакансии Юрий ОТКЛОНИЛ.
- Вместо: при МАССОВОМ отказе — предложить «выберите, кого добавить в Резерв» (хорошие
  кандидаты не теряются). Остальным — обычный отказ.
- Кому добавили в Резерв — уходит ДРУГОЕ сообщение (не «отказ», а «ваша кандидатура у нас в
  резерве, свяжемся при подходящей позиции»), не нейтральный отказ.

## 4. Жизненный цикл / хранение (152-ФЗ)
- «Не подходит» → **Архив** (кандидат сохранён, но помечен «больше не подходит»).
- Архив → авто в **Корзину** через срок (по умолчанию **5 месяцев**), настраивается.
- Корзина: ещё **1 месяц** хранения → авто-удаление.
- Можно настроить «никогда не удалять», НО показываем предупреждение: по 152-ФЗ хранить
  перс. данные без основания некорректно.
- Поля: `archivedAt`, `trashedAt` (+ company-настройка срока, дефолт 5 мес).

## 5. Дожим — объединить legacy + v2 в один (общая задача упрощения).

## Инкременты
1. **Источник + добавление из отборов** (приоритет Юрия): `sourceType` (миграция), чип
   источника в Базе, кнопка «В резерв» у кандидата (sourceType=vacancy_reject + ссылка на вакансию).
2. **Жизненный цикл**: archive/trash поля + действия «Не подходит→Архив», cron авто-перемещения
   (5 мес → корзина → +1 мес → удаление), company-настройка срока + предупреждение 152-ФЗ.
3. **Прогрев**: авто-касания + статусы; массовый отказ → «выбрать в Резерв» + отдельное сообщение.

## Исходящий подбор (ПОЗЖЕ, не связан с Резервом)
Включается, когда откликов мало / нужно больше. Бот ищет резюме на hh (как hh-поиск).
ВАЖНО: перед «открытием» резюме (платный просмотр/квота) — СНАЧАЛА скан/скоринг по сниппету;
открываем только тех, кто ≥ порога (градус, дефолт 75). Открытым — приглашение откликнуться
на нашу вакансию. С Резервом пока НЕ связываем.
