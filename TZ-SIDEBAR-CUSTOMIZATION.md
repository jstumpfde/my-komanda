# ТЗ: Кастомизация сайдбара — настроить видимость модулей и пунктов

## Контекст
Сайдбар содержит 12+ модулей — перегружен. Нужна возможность скрывать целые модули и отдельные подпункты. Для MVP — localStorage, потом перенесём в БД.

## Перед началом
- Прочитай `components/dashboard/sidebar.tsx` (или где рендерятся модули)
- Пойми структуру: модули → подпункты (menuItems)

## 1. Кнопка "Настроить меню"

В самом низу сайдбара (перед "Powered by Company24.pro"), добавить кнопку:
- Иконка: SlidersHorizontal или LayoutGrid из lucide-react
- Текст: "Настроить меню"
- Стиль: text-xs text-muted-foreground hover:text-foreground, компактная

## 2. Панель настройки (Sheet/Drawer справа)

При клике → открывается Sheet (shadcn/ui Sheet) справа:
- Заголовок: "Настройка меню"
- Подзаголовок: "Выберите модули и пункты для отображения"

### Список модулей:
Для каждого модуля:
- Чекбокс + иконка модуля + название
- Если модуль включён — под ним список подпунктов (menuItems) с чекбоксами
- Если модуль выключен — подпункты скрыты (серые)
- Визуально: отступ для подпунктов, разделители между модулями

### Пример:
☑ 👥 HR и команда
☑ Дашборд
☑ Вакансии
☑ Кандидаты
☐ Аналитика        ← скрыт
☑ 📚 База знаний
☑ Дашборд
☑ Материалы
...
☐ 🚛 Логистика         ← весь модуль скрыт

### Кнопки внизу Sheet:
- "Сбросить" (outline) — вернуть всё по умолчанию (все включены)
- "Сохранить" (primary) — применить + закрыть Sheet + toast "Меню обновлено"

## 3. Хранение — localStorage

Ключ: `sidebar-visibility`
Формат:
```ts
interface SidebarVisibility {
  modules: Record<string, boolean>;     // moduleId → visible
  items: Record<string, boolean>;       // "moduleId:href" → visible
}
```

По умолчанию (если нет в localStorage) — всё видимо (true).

## 4. Применение в сайдбаре

При рендере модулей в sidebar:
1. Прочитать `sidebar-visibility` из localStorage
2. Отфильтровать модули где `modules[moduleId] === false`
3. Отфильтровать подпункты где `items[moduleId:href] === false`
4. Рендерить только видимые

Обернуть в хук: `useSidebarVisibility()` который возвращает:
- `visibility` — текущее состояние
- `setVisibility` — обновить
- `isModuleVisible(moduleId)` — проверка
- `isItemVisible(moduleId, href)` — проверка
- `resetToDefault()` — сброс

## 5. Отраслевые пресеты (бонус)

В верхней части Sheet — секция "Быстрая настройка по отрасли":
- Выпадающий select или карточки:

| Отрасль | Включённые модули |
|---------|-------------------|
| 🏥 Клиника / Стоматология | HR, Обучение, База знаний, Бронирование, CRM, Маркетинг |
| 🚗 Автосервис | HR, CRM, Бронирование, Склад, Маркетинг |
| 🏨 Отель / Хостел | HR, Бронирование, CRM, Маркетинг, Задачи |
| 🏗 Строительство | HR, CRM, B2B, Логистика, Склад, Задачи |
| 🏭 Производство | HR, Обучение, Склад, Логистика, CRM, ОКК, Задачи |
| 💻 IT-компания | HR, Обучение, База знаний, CRM, Задачи, Маркетинг |
| 🛒 Ритейл / Магазин | HR, CRM, Склад, Маркетинг, Бот-звонарь |
| 📞 Колл-центр | HR, Обучение, CRM, ОКК, Бот-звонарь |
| 🎓 Обучающий центр | HR, Обучение, База знаний, CRM, Бронирование, Маркетинг |
| ⚙️ Своё | Ручная настройка (текущий режим) |

При выборе отрасли:
- Автоматически включаются нужные модули, остальные выключаются
- Можно доредактировать вручную после выбора пресета
- Toast: "Применён пресет: Клиника"

## Стиль
- Sheet: w-[400px], стандартный shadcn Sheet
- Чекбоксы: shadcn Checkbox
- Модули: font-medium, подпункты: text-sm text-muted-foreground pl-6
- Пресеты отраслей: маленькие карточки в сетке 2 колонки, border, hover:border-primary
- Активный пресет: border-primary bg-primary/5

## НЕ делать
- Сохранение в БД (потом)
- Drag & drop для перетаскивания порядка (потом)
- Разные настройки для разных ролей (потом)
