# ТЗ-2: Карточка hh.ru на табе «Настройки» вакансии

**Цель:** Сделать так, чтобы на табе «Настройки» вакансии отображалась карточка
hh.ru с информацией об интеграции (статус, hh_vacancy_id, кнопка «Отвязать»,
метрики).
**Оценка:** ~10–15 минут.

---

## КОНТЕКСТ

В предыдущей сессии (v28.04) уже был написан код карточки hh.ru на табе
«Настройки» вакансии — примерно строки 2326–2360 в файле страницы вакансии.

**Проблема:** карточка не отображается у пользователя в браузере. Возможные
причины:
1. Условный рендер не срабатывает (vacancy.hhVacancyId vs hh_vacancy_id)
2. Данные не пробрасываются с сервера на клиент (SSR/page.tsx)
3. Компонент не импортирован или не вставлен в JSX-дерево
4. Кеш SSR (старая версия страницы)

**Тестовая вакансия:**
- ID: 3e8d1f6b-b3bf-4f71-8d77-85a3a9344d71
- hh_vacancy_id: 132184135
- URL: https://company24.pro/hr/vacancies/3e8d1f6b-b3bf-4f71-8d77-85a3a9344d71

---

## ЗАДАЧА

### Шаг 1 — Диагностика
Найди код карточки hh.ru на табе «Настройки» вакансии:

  grep -rn "Отвязать" components app --include="*.tsx" -l
  grep -rn "hh_vacancy_id\|hhVacancyId" "app/(modules)/hr/vacancies" --include="*.tsx" -l

Найди файл страницы вакансии (скорее всего
app/(modules)/hr/vacancies/[id]/page.tsx или
components/vacancies/vacancy-settings-tab.tsx или
components/vacancies/vacancy-ai-process-settings.tsx).

Опиши вкратце:
- В каком файле находится код карточки hh
- Какое условие на её отображение
- Откуда приходит значение hhVacancyId в этот компонент

### Шаг 2 — Исправление

Найди корневую причину и почини. Типичные сценарии:

Сценарий A — Условие неверное:
Если в JSX {vacancy.hhVacancyId && <HhCard />}, а в данных приходит
hh_vacancy_id (snake_case), условие ложное. Исправь маппинг — либо в
API-роуте конвертировать в camelCase, либо читать vacancy.hh_vacancy_id.

Сценарий B — Данные не пробрасываются:
Проверь app/(modules)/hr/vacancies/[id]/page.tsx — есть ли в SELECT-запросе
поле hhVacancyId/hh_vacancy_id. Если в схеме db/schema.ts есть, а при чтении
не выбирается — добавь.

Сценарий C — Компонент не вставлен:
Если карточка существует как компонент, но не используется в JSX — вставь её
в таб «Настройки», выше или рядом с «AI-обработка hh-откликов».

### Шаг 3 — Содержимое карточки

Карточка должна показывать:
- Заголовок «Интеграция с hh.ru» с зелёным бейджем «Подключено», если
  hhVacancyId !== null
- hh_vacancy_id (можно как ссылку: https://hh.ru/vacancy/{id})
- Дата подключения (если есть hhConnectedAt или поле создания связки)
- Метрики: количество откликов в БД для этой вакансии (SELECT count
  из hh_responses по vacancyId)
- Кнопка «Отвязать» — вызывает существующий endpoint
  DELETE /api/integrations/hh/vacancies/[id]/unlink или аналогичный

Если карточка с этим контентом уже существует — просто добей недостающее
или исправь рендер.

### Шаг 4 — Проверка

  pnpm tsc --noEmit 2>&1 | head -30
  pnpm lint 2>&1 | head -30

## ВАЖНО
