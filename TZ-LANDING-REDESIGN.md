# ТЗ: Переработка лендинга Company24.pro

## Контекст
Файл: `app/(public)/landing/page.tsx`
Стек: Next.js 16, TypeScript, Tailwind CSS, shadcn/ui
Темы: 3 штуки (тёмная, светлая, тёплая) — все должны корректно работать через CSS-переменные
UI текст: только русский

## Принцип работы
НЕ удалять существующие секции. Новые секции ДОБАВЛЯЮТСЯ рядом. Старые секции, помеченные как "скрыть", оборачиваются в `{false && (...)}` или `className="hidden"` — чтобы можно было вернуть позже.

## ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

### Анимации
- Все новые секции: плавное появление при скролле (Intersection Observer + CSS transform/opacity). Без framer-motion.
- Карточки агентов: пульсирующий зелёный индикатор (CSS animation, keyframes pulse)
- Карточки модулей: hover — translateY(-4px) + shadow
- Карточки ролей: expand/collapse с transition height

### Темы
- ВСЕ новые элементы используют CSS-переменные для цветов (bg, text, border, accent)
- Проверить что все 3 темы (тёмная, светлая, тёплая) отображаются корректно
- НЕ хардкодить цвета (никаких `bg-gray-900` — только `bg-background`, `text-foreground` и т.д.)

### Адаптивность
- Desktop: полная сетка
- Tablet: 2 колонки
- Mobile: 1 колонка, горизонтальные скроллы где нужно

### Порядок секций на странице (итого):
1. Hero (без изменений)
2. **НОВАЯ: Выберите свою роль** (4 карточки)

4. Для вашего бизнеса — 6 карточек (старая)
5. Не CRM. Не ERP. (старая)
6. **НОВАЯ: 15 модулей** (сетка карточек)
7. Платформа — табы HR/Маркетинг/Продажи/Логистика (старая)

9. ~~3 шага~~ → **СКРЫТЬ**

11. **НОВАЯ: Отраслевые решения** (12 карточек)

### Важно
- НЕ удалять код скрытых секций — только `hidden` / `{false && ...}`
- НЕ использовать фейковые данные: никаких "500+ компаний доверяют", "1000+ пользователей"
- Все тексты на русском
- Иконки: Lucide React (уже в проекте)
- Графики/чарты: если нужны — recharts (уже в проекте)
- После всех изменений: убедиться что `pnpm build` проходит без ошибок
- Коммит: `feat: landing page redesign — roles, 15 modules, 50+ agents, industries`
