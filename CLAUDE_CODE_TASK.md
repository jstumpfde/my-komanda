# ЗАДАНИЕ ДЛЯ CLAUDE CODE: Прототип Конфигуратора Company24

## Контекст

Работаем в проекте **my-komanda** (company24.pro).
Стек: Next.js 16.1.6 (Turbopack), TypeScript, Tailwind, PostgreSQL 16.
Прокси для Claude API: `https://claude-proxy.jstumpf-de.workers.dev` (переменная `CLAUDE_PROXY_URL` в `.env`).
Модель: `claude-haiku-4-5-20251001`.

Это **прототип для живой демонстрации 21 апреля 2026**. Цель — показать клиентам и инвесторам: *"Теперь можно настраивать платформу словами"*. Функция «Создать» пока работает в режиме mock (показывает тост «Отчёт запланирован на Q2 2026»), чат и превью-карточка — настоящие.

3. Никогда не упоминай в интерфейсе «Claude», «Anthropic», «OpenAI», «ChatGPT». Везде — «Нэнси» или «Конфигуратор Company24».
4. Используй существующие паттерны кодовой базы my-komanda (смотри сайдбар, темы, шрифты в `/app` и `/components`).

### Шаг 3. Создать компонент чата

Файл: `components/configurator/ConfiguratorChat.tsx`

Клиентский компонент (`'use client'`). Требования:
- Поле ввода внизу, история сообщений сверху, автоскролл вниз при новом сообщении.
- Сообщения клиента справа, ответы Нэнси слева (как в обычных чатах).
- **Парсер ответа:** если в ответе есть блок ```json ... ```, отделить текстовую часть и JSON. Распарсить JSON и отрендерить через компонент `RoutinePreviewCard` (см. Шаг 4).
- Индикатор загрузки во время ожидания ответа (три точки или "Нэнси печатает...").
- Три кнопки-пресета над полем ввода (показываются только если история пустая):
  1. «📊 Настроить отчёт» — вставляет в поле ввода: *"Хочу настроить отчёт по "*
  2. «⚡ Добавить автоматизацию» — вставляет: *"Хочу автоматизацию: "*
  3. «🎯 Настроить цели» — вставляет: *"Хочу настроить цель на "*
- Состояние хранится локально через useState, без бэкенда (на прототипе истории не сохраняются при перезагрузке страницы — это ок).

### Шаг 4. Создать превью-карточку автоматизации

Файл: `components/configurator/RoutinePreviewCard.tsx`

Карточка с тёмной подложкой (согласуй с темой платформы), показывает распарсенный JSON красиво:
- Заголовок автоматизации крупно сверху.
- Описание подзаголовком.
- Секция «Когда запускается» с иконкой часов (Clock из lucide) и значением trigger.value.
- Секция «Откуда данные» — список sources в виде чипов.
- Секция «Куда отправляется» с иконкой (Send из lucide) и channel + destination.
- Строка снизу: «Движок: {model} · Запусков в месяц: {estimated_runs_per_month}».
- Две кнопки: **«Создать автоматизацию»** (основная, градиент или primary-цвет) и **«Изменить»** (вторичная, обводка).
- Клик по «Создать» — показывает тост/alert: *«Спасибо! Автоматизация будет доступна в Q2 2026. Мы записали вас в список первых пользователей.»* Никаких реальных записей в БД на этом этапе.
- Клик по «Изменить» — отправляет в чат новое сообщение от клиента: *"Измени: "* (клиент продолжает диктовать правку).
