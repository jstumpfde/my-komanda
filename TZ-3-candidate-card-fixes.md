# ТЗ-3: UX багфиксы карточки кандидата

**Цель:** Починить три бага в списке и карточке кандидата на /hr/candidates.
**Оценка:** ~15-20 минут.

---

## БАГИ К ИСПРАВЛЕНИЮ

### Баг 1: Имя кандидата не отображается в списке /hr/candidates

**Симптом:** в списке кандидатов отображается «Новый кандидат» вместо реального
имени, хотя имя в БД есть.

**Где искать:**
- API endpoint: app/api/modules/hr/candidates/route.ts (GET)
- Компонент списка: components/candidates/candidates-list.tsx или
  app/(modules)/hr/candidates/page.tsx

**Что сделать:**
1. Проверь SELECT в API — выбираются ли поля firstName, lastName, fullName
   из таблицы candidates
2. Проверь компонент — какое поле он отображает (name, fullName, firstName)
3. Если данные приходят, но не отображаются — исправь маппинг
4. Если данные не приходят — добавь поля в SELECT

**Тест:** После фикса в списке /hr/candidates должны быть реальные имена для
тех 116 откликов которые в БД (имена есть, проверено через psql).

---

### Баг 2: Скролл «Ответов» в карточке кандидата не работает

**Симптом:** В карточке кандидата (Drawer на /hr/candidates → клик по строке)
есть таб «Ответы» с длинным контентом — он не скроллится. Контакты скроллятся
нормально (это уже починено в round 5).

**Где искать:**
- components/candidates/candidate-drawer.tsx
- components/candidates/candidate-answers.tsx или похожий
- Найти таб с ответами кандидата на вопросы анкеты/демо

**Что сделать:**
1. Найди контейнер с ответами — скорее всего там max-height без overflow-y: auto
2. Сравни с табом «Контакты» — там скролл работает (round 5), скопируй паттерн
3. Возможно нужно добавить overflow-y-auto в Tailwind, либо высоту flex-1 +
   min-h-0 в родителе

**Тест:** Открой кандидата с длинным списком ответов — список должен
скроллиться внутри Drawer без скролла всей страницы.

---

### Баг 3: Описание мест работы из hh resume не отображается

**Симптом:** В табе «Опыт» карточки кандидата показываются названия компаний,
позиции, даты — но НЕ показывается description (текстовое описание обязанностей).
Поле есть в hh API в response /resumes/{id} → experience[].description.

**Где искать:**
- components/candidates/hh-resume-info.tsx (создан в Пачке 3, 30.04)
- Секция «Опыт работы» / «Experience»
- Также проверь что lib/hh-api.ts → fetchFullResume сохраняет description в БД

**Что сделать:**
1. Проверь схему таблицы где хранится опыт работы (возможно candidates.full_resume
   как JSONB или отдельная таблица candidate_experience)
2. Проверь что fetchFullResume записывает поле description
3. Проверь что frontend читает и рендерит description (может быть форматирован
   как HTML или markdown — нужен <div dangerouslySetInnerHTML> или ReactMarkdown)
4. Если description есть, но не рендерится — добавь его в JSX

**Тест:** Открой любого из 53 кандидатов с full_resume — в табе «Опыт» под
каждой компанией должно быть текстовое описание обязанностей.
